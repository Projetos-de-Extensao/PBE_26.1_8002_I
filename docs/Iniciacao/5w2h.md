# 🌸 5W2H do Site para Gerenciamento de Estágios para a IBMEC 🌸
## 🎀 Introdução

<p align = "justify">
    O 5W2H é uma ferramenta administrativa  que pode ser utilizada para diferentes contextos dentro de uma organização, como na organização para planejamento estratégico e guia na execução de ações. O 5W2H baseia-se em 7 perguntas <b>(What?, Why?, Where?, Who?, When?, How? e How much?)</b>, com o intuito de esclarecer e organizar ideias. A ferramenta será utilizada como um guia de ações, tornando os objetivos mais evidentes para a equipe, auxiliando na compreensão, organização e tomadas de decisões no projeto.
</p>

## 🎀 Metodologia

<p align = "justify">
    A equipe se reuniu para esclarecimento sobre o tema do projeto, a partir dos pontos debatidos foi possível responder às questões do 5W2H.  
</p>

## 🎀 Questões 5W2H

### 💕 Visão geral

|Questão|Resposta|
|-------|--------|
|***What?***|Desenvolvimento da camada de Back-end (servidor, banco de dados e API) de uma aplicação web voltada para a gestão de estágios. O sistema deve centralizar o fluxo de documentos, a validação de horas e o controle de convênios para todos os cursos da instituição, respeitando as diretrizes pedagógicas específicas de cada um. |
|***Why?***|Para automatizar processos manuais da secretaria de estágios, reduzir erros na verificação de requisitos específicos de cada curso (ex: carga horária mínima, pré-requisitos) e facilitar a comunicação entre aluno, faculdade e empresa concedente.|
|***Where?***|O desenvolvimento será realizado em ambiente remoto/laboratórios da faculdade. A aplicação final será hospedada em servidores em nuvem (Cloud) ou na infraestrutura de TI da própria instituição.|
|***When?***|Durante o semestre letivo atual, seguindo o cronograma da disciplina de extensão.|
|***Who?***|Grupo 1 da turma de PBE 2025.2 8002|
|***How?***| Através da metodologia RUP/UP, utilizando:<br><br>• **Linguagem/Framework:** Python com Django<br>• **Banco de Dados:** Relacional MySQL para garantir a integridade dos dados acadêmicos.<br>• **Arquitetura:** RESTful API para comunicação com o Front-end.<br>• **Lógica de Negócio:** Implementação de um motor de regras que valide as exigências variadas de cada curso cadastrado. |

### 💕 Visão do produto

|Questão|Resposta|
|-------|--------|
|***What?***| Gerência de Regras de Negócio e Documentação de Estágios.|
|***Why?***| Para otimizar o tempo de resposta da coordenação, substituindo a conferência manual de documentos por uma validação sistêmica que assegure a integridade dos dados e o cumprimento dos prazos institucionais.|
|***Where?***|No servidor central da aplicação.|
|***When?***| A gerência será executada em tempo real a cada submissão de contrato ou relatório pelo usuário.|
|***Who?***| O sistema (Back-end) processa, mas a supervisão é dos coordenadores de curso.|
|***How?***| Utilizando o Software para processar as requisições via rotas autenticadas e consultas ao banco de dados.|
|***How much?***|Tempo de processamento computacional e manutenção do código-fonte.|

## 🎀 Conclusão

O 5W2H é um técnica bem simples e de fácil entendimento, sem a necessidade de um especialista. Dentro da disciplina a técnica fornece um entendimento melhor, para a equipe, sobre o projeto como responsabilidades, motivação e organização de ações.
